## What "Cloud" means

The same Feedbot, hosted by us at `app.devfeedbot.dev`. You sign up, create projects, connect Telegram, wire Claude Code, and pay a monthly subscription. We handle:

- Postgres backups and point-in-time recovery.
- Upgrades and migrations.
- Email delivery (no SMTP to configure).
- DDoS, TLS, observability, on-call.
- Data residency (EU-first).

If you'd rather run it yourself, [self-hosting](/getting-started/quickstart/) stays free forever and is **not a feature-stripped version**. The cloud is ops-as-a-service, not a different product.

## What's the difference vs self-host?

The product is identical. The boundary is operational:

| | Self-host | Cloud |
|---|---|---|
| Price | Free, MIT | Paid, monthly |
| Where it runs | Your infra | Our infra (EU) |
| Postgres | You manage it | Managed by us, daily backups |
| Upgrades | You pull, you migrate | Done for you |
| SMTP / email | You configure | Done for you |
| Telegram bot | You bring (one global token) | We provide a shared bot, or you bring your own |
| LLM keys | You bring (encrypted at rest) | You bring (encrypted at rest, same as self-host) |
| SLA | None (it's your box) | Yes, see pricing page |
| Data residency | Wherever you put it | EU-first |

## Roadmap

The cloud depends on milestones already on the [public roadmap](https://github.com/helderpgoncalves/feedbot#-roadmap):

- **M5** — Multi-tenant managed bot infrastructure (managed Telegram + WhatsApp sessions).
- **M6** — Billing, Stripe webhooks, plan-based quotas.

Self-hosting moves first; cloud follows once the multi-tenant story is proven on real installs.
