## What you get

<CardGrid>
  <Card title="Telegram capture" icon="seti:default">
    One bot serves every project. A chat is bound to a project at onboarding; routing is server-side via `chat_id`. No tokens to type.
  </Card>
  <Card title="LLM auto-triage" icon="rocket">
    OpenAI or Anthropic structured outputs fill `type`, `severity`, `summary`, `tags`, `language`, `sentiment`. Per-project, encrypted keys, monthly budget cap.
  </Card>
  <Card title="MCP over HTTP" icon="puzzle">
    Native MCP at `/mcp` — no proxy process. `claude mcp add --transport http` and Claude Code can list, triage, and resolve tickets from your editor.
  </Card>
  <Card title="Conversational loop" icon="comment">
    Replies and resolutions land in the same chat the feedback was reported in. When the reporter answers, the ticket flips back to `triaged` automatically.
  </Card>
  <Card title="Closed by default" icon="approve-check">
    No public sign-up on self-host. Owner created via `/setup`, everyone else by invitation. Argon2id keys, Fernet-encrypted LLM keys, HSTS, CSP.
  </Card>
  <Card title="Boring tech" icon="setting">
    FastAPI · SQLAlchemy 2 async · Postgres · Alembic · Jinja + HTMX + Tailwind. Easy to read, fork, and contribute to.
  </Card>
</CardGrid>
